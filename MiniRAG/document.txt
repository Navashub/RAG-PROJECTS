Retrieval-Augmented Generation (RAG) is a technique that improves the accuracy
of large language models by giving them access to external knowledge at query time.
Instead of relying solely on what the model learned during training, RAG retrieves
relevant documents from a knowledge base and uses them as context when generating
an answer.

The RAG pipeline has two main phases. The first is indexing: documents are split
into smaller chunks, each chunk is converted into a numerical vector (an embedding)
using an embedding model, and these vectors are stored in a vector database.

The second phase is retrieval and generation: when a user asks a question, the
question is also converted into an embedding. The system then searches the vector
database for chunks whose embeddings are most similar to the question embedding.
The top matching chunks are retrieved and added to the prompt sent to the LLM.

Vector similarity is usually measured using cosine similarity, which calculates
the angle between two vectors. A score of 1.0 means the vectors are identical
in direction (very similar meaning), while a score near 0 means the texts are
unrelated.

RAG is widely used in enterprise applications, customer support bots, and
document Q&A systems because it allows organisations to keep their LLMs updated
with new information without expensive retraining. The knowledge base can be
updated simply by adding new documents and re-embedding them.

A common challenge in RAG is chunk size selection. Chunks that are too small
may lose context, while chunks that are too large may include irrelevant information
and dilute the answer. Overlapping chunks (where each chunk shares some text with
the next) help ensure that sentences at boundaries are captured fully.
